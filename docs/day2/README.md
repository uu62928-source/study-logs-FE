# 2日目 ドキュメント案内

2日目のテーマは「型モデリングとコンポーネントAPI設計」です。

TypeScriptの型で仕様やUI状態を表現し、不正な値やpropsの組み合わせを作りにくくする方法を学びます。Domain型、Form型、DTO、ViewModelの境界を、学習ログアプリへ機能を追加しながら整理します。

## ドキュメント

### 1. [学習ガイド](./study-guide.md)

2日目の目的、達成目標、合格条件、7.5時間の進め方をまとめています。

学習を始める前に読み、今日どこまで到達すればよいか確認するためのドキュメントです。

### 2. [学習ノート](./learning-notes.md)

学習中に分かりにくかった点、試した実装、今後も使える判断基準を記録します。

理解が曖昧な点を見つけたときや、Codexとの対話から得た知識を残すときに使います。

### 3. [振り返り](./retrospective.md)

2日目に学んだこと、採用・見送りにした型設計、理解度、次回の課題を記録します。

学習の最後に、自分の言葉で説明できることと、まだ確認が必要なことを整理するためのドキュメントです。
